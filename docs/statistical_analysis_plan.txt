# NeuroCogEEG Statistical Analysis Plan

## Purpose

This document defines the planned statistical analysis strategy for the NeuroCogEEG project.

The goal is to separate:

- primary analysis variables
- secondary/descriptive variables
- quality-control variables
- figure-only descriptive outputs

from each other before running inferential statistics.

This document should be updated before statistical scripts are finalized.

---

## General Principles

All statistical analyses will use the validated SPSS main CSV files generated by:

```bash
python pipelines/run_full_analysis.py
````

Main analysis files:

* `outputs/flanker/csv/flanker_spss_main.csv`
* `outputs/gonogo/csv/gonogo_spss_main.csv`
* `outputs/readysetgo/csv/readysetgo_spss_main.csv`
* `outputs/tmt/csv/tmt_spss_main.csv`

The full result CSV files and QC reports remain available for diagnostic checks, but the SPSS main files are the primary statistical input.

---

## Groups

The between-subject factor is:

* `group`

  * `control`
  * `experimental`

For TMT, there is an additional within-subject/task factor:

* `tmt_variant`

  * `tmt1`
  * `tmt2`

---

## Missing Values and QC Rules

Missing values will not automatically remove an entire participant from all analyses.

Instead, exclusion will be analysis-specific.

Examples:

* If `ern_amplitude_uv` is missing because no usable error-response epoch exists, the participant is excluded only from ERN analysis.
* If `cnv_amplitude_uv` is missing because set-locked minimum trial criteria failed, the participant is excluded only from CNV and set-locked connectivity analyses.
* If TMT `rp_mean_uv` or `pmp_peak_uv` is missing because response-locked minimum trial criteria failed, the row is excluded only from TMT RP/PMP analysis.
* TMT PSD and behavioral variables remain usable when response-locked RP/PMP criteria fail.

Minimum-trial flags must be checked before interpreting corresponding EEG outcomes.

---

## Multiple Comparisons Strategy

Multiple comparisons will not be handled by blindly applying one correction across all variables.

Analyses will be organized into theoretically meaningful families:

* behavioral outcomes
* ERP outcomes
* response-locked motor preparation outcomes
* connectivity outcomes
* PSD outcomes
* QC/descriptive outcomes

Primary outcomes will be defined before inferential testing.

Secondary and exploratory outcomes will be reported separately.

Correction strategy will be chosen per analysis family. Options may include:

* reporting uncorrected exploratory p-values with clear labeling
* false discovery rate correction within a defined family
* planned-comparison interpretation for predefined primary outcomes

The correction method must be explicitly reported with each result table.

# Experiment-Specific Plans

## 1. Flanker

### Design

Between-subject comparison:

* control vs experimental

### Primary behavioral variables

* `correct_rt_mean_ms`
* `accuracy_percent`

### Primary ERP variables

* `n2_amplitude_uv`
* `p3_amplitude_uv`

### Secondary ERP variable

* `ern_amplitude_uv`

ERN may be missing in participants with no usable error-response epochs.

### Response-locked variables

* `rp_mean_uv`
* `pmp_peak_uv`

### Connectivity variables

Connectivity variables may be analyzed as secondary/exploratory outcomes.

Families:

* method: `coh`, `wpli`, `ciplv`
* band: theta, alpha, beta
* pair group: interhemispheric, intrahemispheric left, intrahemispheric right

Connectivity analysis should be interpreted cautiously because of the number of variables.

### Suggested statistical model

For each primary continuous outcome:

* independent-samples group comparison

If assumptions are acceptable:

* independent samples t-test or linear model with `group` as predictor

If assumptions are poor:

* Mann-Whitney U test or robust/permutation alternative

## 4. TMT

### Design

Mixed design:

Between-subject factor:

* `group`

  * control
  * experimental

Within-subject/task factor:

* `tmt_variant`

  * tmt1
  * tmt2

### Important participant-matching issue

TMT1 and TMT2 rows must be matched by participant.

Current `subject_id` values may include variant-specific suffixes such as `...1` and `...2`.

Before repeated-measures or mixed-model statistics, a stable participant identifier must be derived.

A planned derived variable should be created:

* `participant_id`

This variable should identify the same participant across TMT1 and TMT2 rows.

TMT statistics must not proceed until participant matching is validated.

### Primary behavioral variables

* `task_duration_s`
* `error_percent`

### Secondary behavioral variables

* `correct_click_count`
* `error_click_count`
* `inter_click_interval_mean_s`

### Primary PSD variables

ROI-level relative power:

* frontal theta
* frontal alpha
* frontal beta
* parietal-occipital theta
* parietal-occipital alpha
* parietal-occipital beta

Variables:

* `frontal_theta_relative_percent`
* `frontal_alpha_relative_percent`
* `frontal_beta_relative_percent`
* `parietal_occipital_theta_relative_percent`
* `parietal_occipital_alpha_relative_percent`
* `parietal_occipital_beta_relative_percent`

### Early/late PSD variables

Early/late and change variables are secondary or exploratory.

Examples:

* `early_frontal_theta_relative_percent`
* `late_frontal_theta_relative_percent`
* `change_frontal_theta_relative_percent`

### Response-locked variables

* `rp_mean_uv`
* `pmp_peak_uv`

These require response-locked minimum trial criteria.

Rows failing `response_locked_trials_minimum_met` should be excluded from TMT RP/PMP analysis, but not from behavioral or PSD analyses.

### Suggested statistical model

For TMT behavioral and PSD outcomes:

* mixed design model

  * between-subject factor: `group`
  * within-subject factor: `tmt_variant`

Preferred:

* linear mixed model with participant-level random intercept

Alternative:

* mixed ANOVA after reshaping data to wide format

Key effects:

* main effect of group
* main effect of TMT variant
* group × TMT variant interaction

---

## Figure Outputs

Figures are considered reporting and diagnostic outputs.

They do not define statistical significance by themselves.

Completed figure classes:

* CSV-based summary figures
* Flanker waveform figures
* Go/No-Go waveform figures
* ReadySetGo waveform figures
* TMT waveform figures
* TMT descriptive scalp topomaps

TMT scalp topomaps are descriptive. They should not be interpreted as high-density source localization because the headset has limited channel coverage.
